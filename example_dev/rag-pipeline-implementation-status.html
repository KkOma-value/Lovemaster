Step 3: 策略反馈闭环 MVP 实现
从反馈数据到策略评分的闭环，只读洞察不影响线上检索
F1
读取 Feedback
WikiFeedbackEventRepository.aggregateByCandidateAndType()
读取近 N 天(默认14天)的反馈事件
按 candidateId × eventType 聚合
Done@Query JPQL窗口聚合
F2
策略评分
StrategyScoringJob.computeScores()
@Scheduled cron: 0 30 */6 * * *
计算 positive_rate / continue_rate / confidence
UPSERT 到 wiki_strategy_score
Done每6.5hUPSERT

rank_score 公式

rank = positive × 0.6 + continue × 0.3 + confidence × 0.1
positive_rate = helpful / (helpful + unhelpful)
continue_rate = continue_chat / total (暂估: positive × 0.8)
confidence = min(sample_count / 30, 1.0)
F3
策略排序
StrategyRankingService.topStrategies()
按 rank_score DESC 排序
top-K 限制（默认 20）
Done按 topicKey 筛选
F4
灰度判定
StrategyRankingService.shouldGrayEnable()
userId hashCode % 100 < grayPercent
默认 10%，同用户结果稳定
Done一致性 hash默认10%
