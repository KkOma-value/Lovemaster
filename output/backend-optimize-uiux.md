# UIUX: Lovemaster 后端优化 - Dify RAG 集成

> 版本：v1.0
> 日期：2026-03-29

## 1. 概述

本次改动为纯后端优化（RAG 数据源切换），不涉及前端 UI 变更。

本文档记录对用户可感知行为的影响，确保前端体验不回退。

---

## 2. 用户可感知变化

### 2.1 回答质量

| 维度 | 改动前 | 改动后 |
|------|--------|--------|
| RAG 知识来源 | 本地 Markdown 文件 | Dify Cloud 知识库 |
| 检索方法 | 简单余弦相似度 | 混合检索（语义 + 关键词） |
| 知识管理 | 需修改代码重启 | Dify UI 即时更新 |

**预期效果**：回答更准确、更贴合知识库内容，知识更新更及时。

### 2.2 响应延迟

| 场景 | 改动前 | 改动后 |
|------|--------|--------|
| RAG 检索耗时 | ~10-50ms（本地） | ~100-500ms（网络） |
| 总响应首字延迟 | ~1-2s | ~1.5-2.5s |

**影响**：SSE 首字延迟可能增加 100-500ms，但流式输出体验不变。

### 2.3 降级场景

当 Dify 不可用时：
- 用户**不会**看到错误提示
- Brain Agent 会基于自身知识回答（无 RAG 增强）
- 后台日志记录降级事件

---

## 3. SSE 事件流（无变化）

现有 SSE 事件类型保持不变：

```
Chat 模式:
  intake_status → ocr_result → rewrite_result → rag_status → content → done

Manus 模式:
  intake_status → ocr_result → rewrite_result → rag_status
    → [content | tool_call] → done
```

`rag_status` 事件内容可微调为 "正在从知识库检索..."（如果前端有显示的话），但事件类型不变。

---

## 4. 前端无需改动

- 所有 API 端点不变
- SSE 事件结构不变
- 请求参数不变
- 响应格式不变

本次后端优化对前端完全透明。
