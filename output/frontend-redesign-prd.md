# Lovemaster Frontend Redesign PRD

## 1. 目标
彻底重构 Lovemaster Web 客户端前端界面，完全对齐用户提供的现代、温馨、富有情感的设计稿（Glassmorphism + 悬浮卡片式）。解决当前用户反馈的“界面不美观”问题，提升产品整体质感和用户粘性。

## 2. 核心功能与页面需求

### 2.1 基础布局 (App Shell)
- **左侧边栏 (Sidebar)**：
  - 顶部：Logo / “新对话” 按钮。
  - 中部：历史对话列表（需表现选中和未选中状态，颜色柔和）。
  - 底部：用户档案区（头像、昵称、邮箱、退出/设置按钮）。
- **右侧主工作区 (Main Content)**：
  - 动态模糊背景：根据应用主题或随机匹配温馨的情感模糊图。
  - 顶部栏：(可选) 右上角“返回首页”或辅助按钮。

### 2.2 欢迎/初始状态页 (Home/Empty State)
- **视觉中心**：居中的品牌图标（圆形内的爱心）。
- **问候语**：优雅的大号中文字体，“我是你的恋爱助手，今天有什么可以帮忙的？”。
- **快速输入框**：
  - 位于页面中央。
  - 圆角胶囊形状。
  - 包含 `placeholder` 和右侧发送箭头。

### 2.3 分析引导页 (Analysis State)
- **视觉中心**：居中的品牌图标。
- **引导文案**：
  - 主标题：“发聊天截图或对话内容，我先帮你发看懂对方在想什么”。
  - 副标题说明使用场景。
  - 推荐操作按钮（例如“先分析，再给建议”）。
- **复合输入区**：
  - 大尺寸白色悬浮软质面板。
  - 左侧提供明确的“附件上传/图片上传”入口。
  - 右侧为发送按钮。
  - 底部附带隐私或使用说明小字。

### 2.4 对话流态 (Chat Stream - 需脑补以匹配整体风格)
- 当用户发出消息后，界面应向上过渡。
- 消息气泡需保持极简的圆角风格，背景色与主基调保持一致。
- 输入框固定到页面底部。

## 3. 非功能性需求
- **响应式设计**：确保在各种桌面窗口大小和移动端下正常显示，侧边栏可在窄屏时收起。
- **性能**：高斯模糊效果 (`backdrop-filter`) 不能引起页面卡顿，须使用 CSS 硬件加速。
- **无缝体验**：使用 React 的过渡动画（如 Framer Motion）平滑切换首页态和对话态。
